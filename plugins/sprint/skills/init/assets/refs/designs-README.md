# Feature Designs

Design documents for each feature.

## Naming Convention

`F{n}-{feature-name}.md`

Examples:
- `F1-user-auth.md`
- `F2-payment-system.md`

## When to Create

- Before starting implementation of a Feature
- When significant design decisions are needed
- Created automatically by `/sprint:add-backlog` if requested

See: `assets/templates/FEATURE-DESIGN-TEMPLATE.md` in sprint-init skill
