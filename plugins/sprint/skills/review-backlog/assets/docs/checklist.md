# Documentation Review Checklist

> 문서 작성 Task/Feature 검토용

---

## 1. 정확성 (Accuracy)

- [ ] 내용이 실제 구현/상태와 일치하는가?
- [ ] 오래된 정보가 있는가?
- [ ] 누락된 내용이 있는가?
- [ ] 코드 예제가 실제로 동작하는가?
- [ ] 링크가 모두 유효한가?

---

## 2. 명확성 (Clarity)

- [ ] 대상 독자가 이해할 수 있는가?
- [ ] 모호한 표현이 있는가?
- [ ] 전문 용어가 설명 없이 사용되었는가?
- [ ] 예시가 충분한가?
- [ ] 복잡한 개념에 다이어그램/시각 자료가 있는가?

---

## 3. 구조 (Structure)

- [ ] 논리적 흐름이 있는가?
- [ ] 정보를 찾기 쉬운 구조인가?
- [ ] 중복된 내용이 있는가?
- [ ] 제목/섹션이 내용을 정확히 반영하는가?
- [ ] 목차/네비게이션이 적절한가?

---

## 4. 완성도 (Completeness)

- [ ] 정의된 범위를 모두 다루었는가?
- [ ] 시작하기(Getting Started) 가이드가 있는가?
- [ ] 자주 묻는 질문/트러블슈팅이 있는가?
- [ ] 다음 단계/관련 문서 링크가 있는가?

---

## 5. 유지보수성 (Maintainability)

- [ ] 업데이트하기 쉬운 구조인가?
- [ ] 버전/날짜가 명시되어 있는가?
- [ ] 담당자/연락처가 명시되어 있는가?
