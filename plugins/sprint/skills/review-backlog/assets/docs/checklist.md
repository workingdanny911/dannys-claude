# Documentation Review Checklist

> Comprehensive review checklist for documentation Tasks/Features.

---

## 1. Accuracy

- [ ] Does the content match the actual implementation/state?
- [ ] Is there any outdated information?
- [ ] Is there any missing content?
- [ ] Do code examples actually work?
- [ ] Are all links valid?

---

## 2. Clarity

- [ ] Can the target audience understand the content?
- [ ] Are there ambiguous expressions?
- [ ] Are technical terms used without explanation?
- [ ] Are there sufficient examples?
- [ ] Are there diagrams/visuals for complex concepts?

---

## 3. Structure

- [ ] Is there a logical flow?
- [ ] Is the structure easy to navigate?
- [ ] Is there duplicated content?
- [ ] Do headings/sections accurately reflect their content?
- [ ] Is table of contents/navigation appropriate?

---

## 4. Completeness

- [ ] Does it cover the entire defined scope?
- [ ] Is there a Getting Started guide?
- [ ] Is there a FAQ/troubleshooting section?
- [ ] Are there next steps/related document links?

---

## 5. Maintainability

- [ ] Is the structure easy to update?
- [ ] Are version/date indicated?
- [ ] Are owner/contact information specified?
